# 更新日志

HondataDash 从首个上车可用版本到当前版本的所有重要变更。

本 App 是一个面向 Hondata FlashPro 蓝牙数据的轻量 Android 横屏仪表，用于 800×480 车机实时数据显示。它不是通用 OBD 工具，也不能替代 Hondata FlashPro Manager。

---

## V2.7.0 — 主数据语义颜色
*2026-06-24 · versionCode 24*

**概要**
在 V2.6.9 蓝牙生命周期与会话完整性稳定基线之上，为 L.TRIM、MAP、IGN、S.TRIM 增加主文字语义颜色，使驾驶者可以更直观地区分安全、警告和异常区间。

**主要变化**
- L.TRIM / S.TRIM：按绝对值判断，±5% 内绿色，±15% 内黄色，超过 ±15% 红色。
- IGN：0° 及以上绿色，-5° 至 0° 黄色，低于 -5° 红色。
- MAP：按当前主卡片显示的相对增压 bar 判断，1.45bar 内绿色，1.45~1.60bar 黄色，超过 1.60bar 红色。
- ECT：正常工作温度区间（80~95°C）由白色改为绿色，冷车蓝色和过热红色不变。
- 刻度条保持 V2.6.9 原有方向/情绪模式，不做改动。

**保持不变**
- DFCO / SYNC 语义显示逻辑不变。
- A/F Lambda 语义颜色和 WOT 过稀闪烁逻辑不变。
- A/F / IGN / S.TRIM 低置信度灰显逻辑不变，且仍然拥有最终覆盖优先级。
- 蓝牙连接、重连、协议解析、状态机、MAX/MIN 极值记录全部不变。

**实车意义**
- 日常巡航时可以快速判断燃油修正是否健康。
- WOT 或高负荷时可以更直观看到点火角和增压是否进入警告区。
- 保持原有稳定性，只增强驾驶时的可读性。

## V2.6.9 — 蓝牙生命周期与会话完整性修复
*2026-06-18 · versionCode 23*

**概要**
修复 V2.6.8 审查报告的全部关键问题，重点解决切后台误判断线、首次连接失败不重连、长断后极值被覆盖等问题。

**主要变化**
- 切后台（onPause）不再被误判为蓝牙断线，回前台后正常恢复；如果后台期间蓝牙断了，自动重连。
- 首次连接握手失败或异常，现在会自动进入重连循环，不再卡死。
- 长时间断连后发动机还在跑，极值真正保留不被覆盖。
- 协议帧严格校验：不完整的传感器定义或数据帧直接拒绝，不再返回半截数据。
- 关键传感器数据缺失时，置信度降到 0，不再误报"正常且可信"。

**修复问题**
- 重连后首帧 RPM 数据缺失（NaN）时，不再误判发动机在运行。
- 数据帧长度异常时走完整失败流程，不再死锁在"已连接但无数据"状态。
- `connected` 标志改为握手成功后才置 true。
- 蓝牙被用户关掉时停止重连，不再无限重试耗电。
- 握手协议解析失败时清空缓冲重试，不直接放弃。
- `onDestroy` 先解绑回调再断开，避免回调到已销毁的界面。

## V2.6.8 — 稳定性与鲁棒性加固
*2026-06-17 · versionCode 22*

**概要**
防御性加固：重连防竞态、NaN 防护、修复历史遗留 BUG。

**主要变化**
- 蓝牙握手各阶段加超时，不再可能永久阻塞。
- 重连入口加锁保护，防止多线程同时触发重连。
- 全链路 NaN 防护（协议层、原始值、计算值、极值初始化），异常数据不会污染显示。
- 低置信度灰显固定颜色和透明度，不同卡片深浅一致。
- WARMUP 暖机退出不再依赖闭环标志，L15B7 冷启快速进闭环不会导致暖机状态卡住。
- 极值可信范围与显示范围校验统一。

**修复问题**
- 首次启动时不再出现假爆震黄色闪烁（ECU 累积值被误判为新增爆震）。
- 闪烁定时器在界面销毁时正确清理，不再内存泄漏。
- 蓝牙被系统关闭时停止重连风暴。
- 握手协议错位时清空缓冲重试，而非直接失败触发完整重连。
- 长断重连后如果发动机还在跑，正确重置基准标志。

## V2.6.7 — 发动机基准、蓝牙重连加固
*2026-06-15 · versionCode 21*

**概要**
V2.6.6 之后的防御性收口：取消启动自检，给 MAX/MIN 一个点火时的已知良好基准，短时蓝牙断连下保留数值层，并修复重连路径静默死亡的问题。

**主要变化**
- 取消正式版启动自检扫掠；`onCreate` 现在直接进入连接流程。
- 加入一次性发动机运行基准覆盖：引擎启动时，每张卡片从首个有效样本获得 MAX/MIN 基准（`engineBaselineApplied[8]`），按卡片门控——乙醇需先通过稳定门控。引擎熄火后清除标志，下次启动重新基准化。
- 短时蓝牙断连（5 分钟内）保留数值与极值层；仅当断连超过 5 分钟且 RPM ≤ 300 时才重置会话。
- 置信度灰色细化：阈值 0.82→0.78，固定颜色 `0xFF888888` + 满 alpha（取消 alpha 渐变）。暖机与动态低参考价值检测不再依赖 `closedLoop`。

**修复问题**
- 握手失败不再设置 `intentionalDisconnect`（此前会阻断自动重连）。
- 轮询循环捕获 `RuntimeException`，线程不再静默死亡而不触发重连。
- RFCOMM 释放等待 1s→2.5s，适配旧版安卓蓝牙栈释放慢的问题。
- 资源释放统一收敛到 `closeAllBluetoothResources` / `cleanupFailedConnection` / `scheduleReconnect` / `closeQuietly`。

**实车意义**
- 极值从真实读数起步，而非点火后的第一个噪声字节。
- 一次短暂蓝牙断连不再丢掉整个行驶周期的 MAX/MIN。
- 握手或解析失败后重连能正常发生。

**备注**
- 状态：待实车验证。

## V2.6.6 — 自检隔离、置信度灰显与乙醇稳定门控
*2026-06-11 · versionCode 20*

**概要**
在 V2.6.5 验证基线之上，对极值与低置信度显示的三项质量修复。

**主要变化**
- 通过 `resetAllExtremeHistory()` 在自检扫掠后清除，将启动自检值与正常极值记录隔离。
- 引擎相关极值（L.TRIM、MAP、A/F、IGN、S.TRIM）按发动机运行周期重置（`engineExtremeSessionActive`）。
- A/F、IGN、S.TRIM 在置信度低于 0.82 时灰显，由 `EngineSemanticState.textAlpha()` 驱动，alpha 下限 0.70。
- 乙醇极值需 3s 最小观察 + 1.5s 稳定（delta < 0.3%）后才记录，修复 CANFlex 0→真实值爬升导致的 MIN=0。

**实车意义**
- 自检不再污染会话极值。
- 乙醇 MIN 不再在重新连接后显示虚假的零。

**备注**
- 这是自检"隔离"尝试。V2.6.7 后续整体移除了自检，因此本版描述的是中间状态，不是最终方案。

## V2.0 — ECU 工况语义状态引擎
*2026-05-24 · versionCode 5*

**概要**
把状态识别从简单节气门判断改为基于 ECU 策略信号，并加入置信度维度。

**主要变化**
- `EngineSemanticState` 携带四个正交维度：MainState、SubState/Modifier、Confidence。
- MainState（互斥）：DFCO / WOT / WARMUP / IDLE / NORMAL。
- WOT 针对扭矩模型 ECU（L15B7）重新定义：开环 + 浓命令 + 增压建立 + RPM 阈值——与节气门位置无关。
- 置信度（0.0–1.0）驱动文字透明度与滤波激进程度。

**实车意义**
- WOT 不再受节气门位置影响而误判，这对扭矩模型 ECU（节气门 ≠ 负荷）很重要。
