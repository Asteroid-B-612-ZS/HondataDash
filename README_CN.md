# HondataDash

面向 Hondata FlashPro 蓝牙数据的轻量 Android 横屏仪表，用于 800×480 车机上的实时数据显示。它关注实车观察中最常用的一组调校相关参数，重点是在车内快速、稳定地看数据，而不是完整的 ECU 标定。

## 这个 App 是什么

- 一个紧凑、低延迟的仪表，通过蓝牙 SPP 读取 Hondata FlashPro 数据。
- 围绕**工况语义显示逻辑**构建：DFCO、暖机、同步恢复、瞬态等工况下的显示值不容易被误认为稳定调校数据。
- 零依赖：纯 Android Framework API，无 `androidx`、无 Kotlin、无 `.so`。正式版 APK 约 52 KB。

## 这个 App 不是什么

- 不是通用 OBD 读取器。只支持 Hondata FlashPro 蓝牙协议。
- 不能替代 Hondata FlashPro Manager。不能刷写或编辑标定。
- 不是自动调校工具。它只显示数据，调校判断由用户完成。

## 目标使用环境

| 项目 | 规格 |

### 6-LED 转速灯条

- 6 颗 LED，按 RPM 递进点亮。
- 阈值（针对 1.5T 动力区间调校）：3000 / 3500 / 4000 / 4500 / 5000 / 5500。
- 颜色：绿色 ×2 → 黄色 ×2 → 红色 ×2。
- ≥ 6400 RPM：全亮，10 Hz 闪烁。
- 与顶栏在 `FrameLayout` 中叠加，节省纵向空间。

### 4×2 主数据网格

每张卡片包含：标签 + 单位（顶部）、大号主值（中左）、实时 MAX/MIN（中右）、刻度条（底部）。

**第一行 — 慢数据**

| 卡片 | PID | 标签 | 格式 | 范围 | 颜色 / 闪烁 |
|------|-----|------|------|------|-------------|
| 0 | 0xB03 | Ethanol | %.0f | 0–100% | <E20 白, E20–40 绿, E40–60 黄, >E60 红 |
| 1 | 0x160 | ECT | %.0f | 40–120°C | <80 蓝, 80–95 白, 96–100 红, >100 紫闪 |
| 2 | 0x151 | IAT | %.0f | 20–100°C | <35 绿, 35–44 白, 45–54 黄, 55–64 红, ≥65 紫闪 |
| 3 | 0x332 | L.TRIM | %+.1f | −25…+25% | DFCO 期间冻结 |

**第二行 — 快数据**

| 卡片 | PID | 标签 | 格式 | 范围 | 颜色 / 闪烁 |
|------|-----|------|------|------|-------------|
| 4 | 0x110 | MAP | %+.1f | −1.0…2.0 bar | 相对压力 (MAP − Baro)，kPa÷100 → bar |
| 5 | 0x320 | A/F | %.1f | 9–18 | Lambda×14.7；<11 红, 11–14.5 黄, 14.5–15.5 绿, >15.5 红 + 给油闪烁；减速断油显示 "DFCO" |
| 6 | 0x140 | IGN | %+.1f | −40…40° | 减速断油显示 "DFCO" |
| 7 | 0x330 | S.TRIM | %+.1f | −25…+25% | 减速断油显示 "DFCO" |

> **A/F 闪烁排除**：A/F 红闪只在节气门 (TP) > 5% 时触发，滑行、松油、换挡不会误报警。

## 数据来源

- **蓝牙（正式）**：通过 SPP 连接 Hondata FlashPro，50 Hz 轮询。目标设备 MAC 在 `BluetoothSource` 中配置。
- **Demo**：`DemoSource` 以 20 Hz 回放真实 LOG，30 秒状态循环（IDLE → NORMAL → WOT → TRANSIENT → DFCO → CRUISE），用于无车环境下测试显示逻辑。

## 工况语义显示

`EngineSemanticState` 携带四个正交维度：

- **MainState**（互斥）：`DFCO` / `WOT` / `WARMUP` / `IDLE` / `NORMAL`。
- **SubState**（细节）：WOT → SPOOL / PEAK / HOLD；DFCO → ENTER / HOLD；或 NONE。
- **Modifier**（正交叠加）：TIP_IN / TIP_OUT / BOOST_SURGE / RPM_DIP / NONE。
- **Confidence**（置信度 0.0–1.0）：加权计算 + 低通滤波（约 200 ms 惯性）。

WOT 由 ECU 策略信号判定（开环 + 浓命令 + 增压建立 + RPM 阈值），而不是节气门位置——这对 L15B7 扭矩模型 ECU（节气门 ≠ 负荷）很重要。

**DFCO 处理**：减速断油期间 A/F、IGN、S.TRIM 灰显 "DFCO"；L.TRIM 冻结在最后一个有效值。进出过渡同步。

## MAX/MIN 记录逻辑

- **极值准入**：只有符合工况条件的样本才能进入极值记录，DFCO 退出、暖机、瞬态尖峰不会污染调校相关的 MAX/MIN。
- **行驶周期 vs 近期**：行驶周期极值与近期动态极值分开记录。
- **发动机基准**（V2.6.7）：引擎启动时，每张卡片从首个有效样本获得一次性 MAX/MIN 基准，按卡片门控（乙醇需先通过稳定门控）。引擎熄火后清除标志。
- **低参考价值灰显**：A/F、IGN、S.TRIM 在置信度低时显示固定灰色，冷启动/暖机读数被视觉标记为低参考价值。

## 蓝牙断连恢复

断线时数据层重启，UI 保持存活：

1. 停掉旧的轮询线程，关闭 socket/流。
2. 等待蓝牙栈释放 RFCOMM channel（V2.6.7 为 2.5 s，较旧版 1 s 提升，适配旧版安卓栈）。
3. 在新线程上从头连接，执行完整握手。
4. 重复失败时指数退避（1 s → 2 s → 4 s → 8 s）。
5. **会话保持**（V2.6.7）：短时断连（5 分钟内）保留数值与极值层；仅当断连超过 5 分钟且 RPM ≤ 300 时才重置会话。

## 安装方式

GitHub Release 不提供 APK（正式版构建包含私有设备 MAC）。请本地构建——见下文——并安装生成的 APK。

## 构建方式

```bash
JAVA_HOME="~/.jdks/jbr-17.0.14" ./gradlew assembleRelease
adb install -r app/build/outputs/apk/release/app-release.apk
```

如需 Demo 构建（LOG 回放，无需车辆），将 `MainActivity` 中 `USE_DEMO` 设为 `true` 后重新编译。

## 使用限制

- 固定 800×480 布局，不适配其他屏幕尺寸。
- 只支持 Hondata FlashPro 蓝牙协议。
- 正式版设备 MAC 硬编码；如需连接你自己的 FlashPro，修改 `BluetoothSource.FLASHPRO_MAC`。
- 无配置界面；调校相关阈值为代码中的常量。
